# Roadmap

1. Manual NotebookLM selector capture session with a dedicated test notebook.
2. Optional adapter that consumes main-repo C2 `SelectorStore` directly.
3. Host-runtime integration that passes explicit run/task IDs into event emission.
4. SP-A/SP-G ingestion helpers for batch source preparation.
5. SP-B consumption examples using strict SourceRecord lists.
6. Optional future MCP/CLI wrappers as separate packages, not inside SP-H core.

Deferred by design: auth/profile management, cookies, undocumented Google APIs, RAG replacement, connector hub, and broad generated artifact downloads.
