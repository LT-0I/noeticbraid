## Top 3 改动建议（按重要性）

1. **Bind the 6 core schemas to runtime OpenAPI (or static-serve frozen YAML).** v1.0.0 promises 13 component schemas but the live FastAPI `/openapi.json` only emits 7. Either pass `Task, RunRecord, SourceRecord, ApprovalRequest, SideNote, DigestionItem` via `extra_models=[...]` on at least one route, or generate a custom `app.openapi_schema` that injects them, or mount `phase1_1_openapi.yaml` statically per Stage 0 v2 line 113. Then add a test asserting `set(openapi['components']['schemas']) == set(ALL_SCHEMA_NAMES)`. This is the only finding that affects external contract semantics. (Dimension #2)

2. **Tighten the WAL/busy_timeout test against Python 3.13's new default.** `test_token_store_initializes_sqlite_schema_with_wal_and_busy_timeout` would pass even if `TokenStore.connect` omitted the `PRAGMA busy_timeout=5000` line, because Python 3.13's `sqlite3` defaults to 5000ms. Strengthen the test, or add a comment that documents the Python-version-dependent assumption. (Dimension #16)

3. **Add CORS preflight smoke test and decide on response-wrapper strictness.** Two non-blocking nits: (a) a single OPTIONS-preflight test would lock CORS dim #17 and keep Lane C unblocked; (b) record the intentional `extra="forbid"` decision somewhere durable so v1.1.0 doesn't accidentally relax it (Dimension #6, MINOR). (Dimensions #17, #6)

## 总体裁决

PROCEED

理由：The artifact is structurally clean — 7 routes, 28 files, all under `packages/noeticbraid-backend/**`, no edits to core/console/contracts/root, raw `sqlite3` only, ctypes-only DPAPI, full SPDX coverage, dependencies on the whitelist, and the claimed `14 passed` pytest run is reproducible (Python 3.13, win32, 0.96s wall). The only finding that could matter for downstream lanes is the 13-vs-7 OpenAPI schema gap, but it is a fix-forward problem, not a contract violation in the YAML, and Stage 0 v2 explicitly anticipated this by recommending static OpenAPI serving as an alternative. All other findings are MINOR and stylistic. Stage 1 is a skeleton, not a freeze, and these gaps can land in a small follow-up patch before Stage 1.5.

## 风险列表（可选）

- **Schema invisibility risk (HIGH).** As long as `/openapi.json` lacks the 6 core schemas, Lane C codegen and any third-party SDK generator cannot consume them through the live API. They must read `phase1_1_openapi.yaml` directly out-of-band. Document this clearly so Lane C does not assume `/openapi.json` is the source of truth.
- **Python 3.13 `sqlite3` default drift (LOW).** If a future Python rolls back `busy_timeout` to 0, the existing test will fail spuriously rather than catch a real regression. Tie the test to behavior of `TokenStore.connect()`, not to the file's persisted state.
- **Lane B / C interface assumptions (LOW).** `storage/factories.py` calls `noeticbraid_core.ledger.RunLedger(root=root)` and `noeticbraid_core.source_index.FileBucketSourceIndex(root=root)`. If Lane B renames `FileBucketSourceIndex` to `SqliteSourceIndex` (per Stage 0 v2 line 273) without keeping a backward-compat re-export, Stage 1's factories will break at runtime. Stage 1 currently has no test that exercises these factories — by design, since core may be absent in CI — but the ImportError will only surface in Stage 2 integration. Recommend Lane B keep both names exported during the transition.
- **Contract document divergence risk (LOW).** Stage 1's `extra="forbid"` plus all-fields-required for the 7 response wrappers is silently stricter than v1.0.0. If Lane B/C tests round-trip JSON through these models, an extra field added by a future v1.1.0 will fail validation at the wrapper layer. Mark the wrapper strictness as Stage 1 -only or carry it forward intentionally into v1.1.0.
- **CI execution path mismatch (LOW).** Local sandbox sets `PYTHONPATH=packages/noeticbraid-backend/src` to run pytest, but each test file pre-pends `SRC_ROOT` itself (`test_*.py:9-12`). On a CI without `pip install -e`, the tests still pass — verified. But if Lane A later removes the per-file `sys.path.insert`, CI must `pip install -e packages/noeticbraid-backend` to make imports work. Note this in `shared_changes.md` proposal #3.
